Краткая инструкция для логиста
  1. Выберите филиал, плановую дату отгрузки с завода, количество дней в пути и целевой срок хранения МЗ.
  2. Оставьте метод «Сезонный спрос», если нужно учитывать месяц поставки. Метод «Средний спрос» используйте только для сверки с прежней логикой.
  3. Загрузите два файла: «Анализ материальных запасов» и «Планы на отгрузку с завода».
  4. Нажмите «Рассчитать потребность».
  5. Если появился блок «Проверьте входные данные», сначала проверьте выбранный филиал, даты и загруженные отчёты.
  6. Основной рабочий столбец — «К заказу, шт.». При необходимости количество можно скорректировать вручную прямо в таблице.
  7. После проверки скачайте заявку в формате XLSX или CSV.
Логика расчёта

1. Отбор данных

  • В расчёт попадают только SKU гардеробной системы из встроенного справочника.
  • Остатки и товар в пути берутся из отчёта «Анализ материальных запасов».
  • Плановые отгрузки берутся из файла «Планы на отгрузку с завода» только по выбранному филиалу.

2. Спрос

  • Базовый спрос считается отдельно по каждому сочетанию филиал × SKU, потому что потребность филиалов отличается.
  • В режиме «Сезонный спрос» базовый спрос умножается на сезонный коэффициент месяца. Если период захватывает два месяца, программа считает спрос по дням каждого месяца.
  • В режиме «Средний спрос» сезонный коэффициент не применяется.

3. Целевой МЗ

Сезонный метод: целевой МЗ = сумма дневного сезонного спроса с даты прихода на выбранный срок хранения.
Средний метод: целевой МЗ = среднемесячный спрос ÷ 30,44 × целевой срок хранения.

4. Итоговая потребность

Расчётная потребность = целевой МЗ + прогноз спроса до прихода − свободно с подтверждением − в пути − планы до выбранной даты.
  • Планы до выбранной даты уменьшают заявку, потому что должны быть отгружены раньше текущей поставки.
  • Планы с выбранной даты и позже показываются справочно и не уменьшают заявку.
  • К заказу, шт. = расчётная потребность, округлённая вверх до кратности упаковки.
